# Rolestra 패키징 가이드 (R11-Task12)

## 산출물

| OS      | Target                | 파일명 패턴                                    | 서명 상태 |
| ------- | --------------------- | ----------------------------------------------- | --------- |
| Windows | NSIS installer (.exe) | `Rolestra-<version>-windows-x64.exe`            | unsigned  |
| macOS   | DMG (.dmg)            | `Rolestra-<version>-macos-{arm64,x64}.dmg`      | unsigned  |
| Linux   | AppImage (.AppImage)  | `Rolestra-<version>-linux-x86_64.AppImage`      | —         |

`directories.output` 은 `dist/electron/` (gitignored).

## 로컬 빌드

```bash
# 현재 OS 만
npm run package

# 특정 OS (cross-build 는 OS 별 GH Actions runner 권장)
npm run package:win
npm run package:mac
npm run package:linux
```

`prepackage` 는 별도 hook 없이 각 스크립트가 `npm run build &&` 로 electron-vite 번들을 직접 갱신한다.
`better-sqlite3` 는 electron-builder 가 `@electron/rebuild` 로 자동 재빌드 (`npmRebuild: true`).

WSL 에서 Linux AppImage 산출은 가능. Windows / macOS 산출물은 GH Actions runner 가 책임진다 — `release.yml` 워크플로우 참조.

## 아이콘

`assets/icon.{png,ico,icns}` 는 placeholder 다.
`tools/assets/build-icons.mjs` 가 1024×1024 PNG 를 procedural 하게 그린 뒤 `png2icons` 로 ICO/ICNS 를 파생한다.

```bash
npm run icons:build
```

브랜드 확정 시 `tools/assets/build-icons.mjs` 를 갱신하거나 디자이너 산출물로 `assets/icon.png` 을 교체하고 위 명령으로 다시 만든다.

## release.yml 트리거

| 트리거 형태             | 동작                                                  |
| ----------------------- | ----------------------------------------------------- |
| `push tag v*`           | 3 OS matrix build → artifact upload → Release draft   |
| `workflow_dispatch`     | 3 OS matrix build → artifact upload (Release 미생성)  |

Release draft 는 `gh release create --draft` 로 생성되고 `dist-artifacts/` 의 모든 파일이 첨부된다 (`--clobber` 로 재실행 안전).

## macOS Gatekeeper 우회 안내

R11 macOS DMG 는 **unsigned + un-notarized** 상태다 (코드 사인 + 공증 → R12+ 이연).
첫 실행 시 macOS Gatekeeper 가 "확인되지 않은 개발자" 경고로 차단한다.
사용자 우회 절차는 두 가지:

1. **Finder right-click** — `Rolestra.app` 우클릭 → "열기" → 경고 다이얼로그에서 다시 "열기" 선택.
2. **CLI** — `spctl --add /Applications/Rolestra.app` 로 화이트리스트에 추가하거나 `xattr -dr com.apple.quarantine /Applications/Rolestra.app` 으로 quarantine 속성 제거.

Apple Silicon arm64 / Intel x64 두 dmg 가 별도 산출되며 사용자 머신 아키텍처와 일치하는 빌드를 다운로드해야 한다 (universal binary 는 R12+).

## 코드 사인 / 공증 / 자동 업데이트 (R12+)

본 phase 에서 **다루지 않는 항목**:

- Apple Developer ID + notarytool 공증 (macOS)
- EV Code Signing 인증서 (Windows SmartScreen)
- `electron-updater` (자동 업데이트 채널 + Release feed)

이 모두 R12+ 의 패키징 라운드 2 에서 다룬다.
