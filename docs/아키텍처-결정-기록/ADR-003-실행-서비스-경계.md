# ADR-003: 파일/명령 실행은 ExecutionService 경계로 강제한다

- 상태: 승인
- 날짜: 2026-02-13
- 결정자: 프로젝트 팀
- 관련 문서: `docs/설계 보완안 (취합).md`, `docs/설계-문서.md`

## 배경

AI가 파일/명령에 접근하는 앱에서 실행 경계가 없으면 보안/감사/복구가 취약해진다.

## 결정

- 실제 반영은 `ExecutionService`를 통해서만 수행
- 반영 순서 고정:
  - `dry-run diff -> 사용자 승인 -> atomic apply -> rollback`
- 명령 실행은 구조화 요청(`command`, `args`, `cwd`)만 허용
- 정책(`allowedCommands`, `blockedPatterns`, `maxExecutionTimeMs`, `maxOutputBytes`) 검증 필수

## 근거

- 권한 경계를 코드로 강제 가능
- 실패 복구와 감사 로그를 일관되게 구현 가능

## 영향

- 긍정: 보안/추적성/운영 안정성 향상
- 부정: 개발 초기 진입 비용 증가
- 마이그레이션 영향: 감사 로그 스키마 및 실행 작업 ID 체계 필요

## 대안

- 각 모듈에서 직접 파일/명령 실행: 구현은 빠르지만 통제 불가.
