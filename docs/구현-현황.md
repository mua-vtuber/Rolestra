**R12-C 1차 종결 사정** (사무실 비유):

원래 R12-C plan 은 21 단계 (T0~T20) 로 짜여 있었음. T11 까지 land 해보니 *직원이 회의에서 합의에 도달하는 진행 절차* (이전 12 단계 합의 진행) 가 새 사무실 구조 (부서별 회의실 + 직원 능력 분리) 와 잘 안 맞음. 사용자 + 직원들 토론 결과 — 진행 절차를 통째로 바꾸기로 결정 (의견 카드 트리 + 일괄 동의 투표).

이 결정이 T12 이후의 작업 정의 (회의 흐름 / 합의 인식 / 부서별 화면) 를 광범위하게 흔들기 때문에, *T0~T11 + 정리 + 멤버 표시 fix* 까지를 **1차로 끊어 main 에 정식 land**, *회의 진행 재설계 본체* 는 **R12-C2 라는 새 phase 에서 처음부터 다시** 진행하는 게 회귀 분리 + 안정 측면에서 안전.
