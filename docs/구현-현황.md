## 1. 현행 기능 (구현 완료)

### 1.1 IPC 시스템
- `IpcChannelMap` 40+ 채널 정의 (`src/shared/ipc-types.ts`)
- `IpcMeta` (requestId, schemaVersion, sequence, timestamp)
- zod 검증 (개발 모드), 스키마 버전 체크, 구조화 에러 (`src/main/ipc/router.ts`)
- `typedInvoke()` + `typedOn()` preload 래퍼 (`src/preload/index.ts`)
- 16개+ push 이벤트 (`src/shared/stream-types.ts`): stream:token, message-start, message-done, state, error, log, turn-wait, consensus-update, execution-pending, permission-pending, failure-report, deep-debate, consensus-document, session-update, mode-transition-request, worker-selection-request, review-request, **cli-permission-request** (신규)
- `provider:list-embedding-models` IPC 채널 추가 (임베딩 전용 모델 목록 조회)
- `cli-permission:respond` IPC 채널 추가 (CLI 권한 요청 승인/거부)

### 1.2 Provider 시스템
- **추상 베이스**: `BaseProvider` (`src/main/providers/provider-interface.ts`)
  - capabilities: Set\<ProviderCapability\>, status listener, warmup/cooldown, streamCompletion, adaptMessages
- **API Provider**: `src/main/providers/api/api-provider.ts`
- **CLI Provider**: Claude Code, Gemini CLI, Codex CLI (`src/main/providers/cli/`)
  - `cli-permission-parser.ts`: Claude Code `permission_request` JSON 파싱 및 `permission_response` 생성
  - `cli-permission-handler.ts`: 대기 중인 권한 요청 Map 관리, IPC 응답 핸들러
  - `cli-stream.ts`: 권한 요청 시 hang timer 일시 중지
- **Local Provider**: Ollama/로컬 LLM (`src/main/providers/local/local-provider.ts`)
- **Registry**: 싱글톤 `ProviderRegistry` (`src/main/providers/registry.ts`)
- **Factory**: config → provider 인스턴스 (`src/main/providers/factory.ts`)
- **DB 영속화**: `ProviderRepository` + `provider-restore.ts`
- **동적 모델 레지스트리**: `model-registry.ts` — CLI 정적 목록 + API 실시간 조회(폴백 포함) + Local 동적 조회

### 1.3 대화 엔진
- **ConversationSession** (`src/main/engine/conversation.ts`): 메시지 이력, 브랜치(fork/switch), 턴 위임, 심층 토론 상태
- **TurnManager** (`src/main/engine/turn-manager.ts`): 라운드 로빈, 라운드 설정, 일시정지/재개/중지/인터럽트
- **Orchestrator** (`src/main/engine/orchestrator.ts`): SSM 상태별 턴 루프 (runArenaLoop 패턴), 사용자 입력 대기/재개 (`waitForUserAction` 전 `state:idle` emit으로 ThinkingIndicator 제거), 2초 간격, 스트리밍, AbortController, 메모리 주입, 심층 토론, EXECUTING 후 작업 요약 파일 경로 보관 → REVIEWING 리뷰어 프롬프트에 전달
- **멀티파티 메시지 변환** (`src/main/engine/history.ts`): 각 AI 관점에서 자기=assistant, 타인=user `[이름]:` 접두사, 연속 user 메시지 병합
- **PersonaBuilder** (`src/main/engine/persona-builder.ts`): 시스템 프롬프트 구성

### 1.4 세션 상태 머신 (SessionStateMachine)
- **12개 상태**: CONVERSATION, MODE_TRANSITION_PENDING, WORK_DISCUSSING, SYNTHESIZING, VOTING, CONSENSUS_APPROVED, EXECUTING, REVIEWING, USER_DECISION, DONE, FAILED, PAUSED (`src/main/engine/session-state-machine.ts`)
- **16개 이벤트**: ROUND_COMPLETE, USER_APPROVE_MODE, USER_REJECT_MODE, SYNTHESIS_COMPLETE, ALL_AGREE, DISAGREE, USER_SELECT_WORKER, WORKER_DONE, REVIEW_COMPLETE, USER_ACCEPT, USER_REWORK, USER_REASSIGN, USER_STOP, USER_PAUSE, USER_RESUME, TIMEOUT, ERROR
- 대화 모드 판단 수집 (`mode-judgment-collector.ts`): 라운드별 AI 판단 (conversation/work) → 과반수 work 시 모드 전환
- 스냅샷 영속화, 단계 타임아웃, aggregator 선택 (designated only, 설정에서 AI 지정)
- 투표 수집 + ABSTAIN 폴백 (`decision-collector.ts`)
- 투표 결과 평가 (`consensus-evaluator.ts`)
- 권한 사이드 이펙트: EXECUTING 진입 시 grant_worker, REVIEWING 진입 시 revoke_worker, 종료 시 revoke_all
- 구조화 메시지 프로토콜: `MessageFormatter` (JSON 포맷 인스트럭션), `AppToolProvider` (상태별 도구 제공)
  - **JudgmentReason enum**: `'code_change' | 'execution_needed' | 'further_discussion' | 'no_action'` — 자유 문자열 대체, 파서 엄격 검증
  - **EXECUTING 포맷 인스트럭션**: Worker에게 합의 폴더에 `work-summary-{timestamp}.md` 작성 지시
  - **REVIEWING 포맷 인스트럭션**: 요약 파일 경로 주입으로 리뷰어가 문서 참조하여 리뷰
- CLI 권한 어댑터: `CliPermissionAdapter` + `respawnWithPermissions()` (worker/read-only 모드 전환), `--add-dir` 플래그로 합의/프로젝트 폴더 사전 허가
- **레거시**: `ConsensusStateMachine` (`consensus-machine.ts`) — `@deprecated`, SSM으로 대체됨

### 1.5 실행 서비스 (ExecutionService)
- **CommandRunner** (`src/main/execution/command-runner.ts`): allowlist, blockedPatterns 정규식, `execFile` + `shell:false`, timeout, maxBuffer

- **AuditLog** (`src/main/execution/audit-log.ts`): 감사 로그 기록

### 1.6 파일/워크스페이스 시스템
