# AI Chat Arena v2 — Definition of Done

작성일: 2026-02-13

## 공통 DoD (모든 작업/PR 필수)

1. 요구사항 일치
- 변경 내용이 `spec/design/설계 보완안` 중 최소 1개 문서와 명시적으로 연결된다.

2. 타입/테스트/린트
- `typecheck` 통과
- `lint` 통과
- 해당 변경 범위 테스트 통과(신규 로직이면 테스트 추가)
- 다음 영역 변경 시 회귀 테스트 필수:
  - 합의 상태머신(`consensus-machine`)
  - IPC 채널/페이로드(`IpcChannelMap`, `IpcMeta`, typed invoke)
  - 권한/실행 경계(`permission-service`, `ExecutionService`)

3. 보안/실행 규칙 준수
- 파일 변경은 `ExecutionService` 경유 설계와 충돌하지 않는다.
- 셸 문자열 실행을 도입하지 않는다.
- 경로 검증/권한 모델을 우회하지 않는다.
- API 키/토큰 등 민감정보가 로그, DB 평문, 에러 메시지, 코드에 노출되지 않는다.
- 민감정보는 반드시 `secrets` 계층(safeStorage)을 통해 관리한다.

4. IPC 규칙 준수
- 문자열 리터럴 직접 호출 대신 typed IPC 계층 사용
- 채널/페이로드 변경 시 타입 및 문서 동시 갱신

5. i18n 규칙 준수
- 사용자 노출 문자열 하드코딩 금지
- 번역 키 추가 시 `ko.json` 동시 반영

6. 마이그레이션 규칙 준수
- forward-only
- idempotent
- 실패 시 startup blocking 정책과 충돌 없음

7. 문서 갱신
- 설계/동작이 바뀌면 관련 문서 갱신
- 큰 결정은 ADR 추가

## Phase별 추가 DoD

## Phase 0
- main/renderer/preload 분리 실행 확인
- typed IPC ping/pong 동작 확인
- i18n 기본(`ko`) 동작 확인

## Phase 1
- 멀티 participant 대화 + 스트리밍 정상
- provider 장애 처리(UI 알림/복구) 확인
- provider adapter 변경 시 `adaptMessages()` 회귀 테스트 통과

## Phase 2
- 합의 상태머신 전이 테스트 통과
- `dry-run -> 승인 -> apply -> rollback` 흐름 검증

## Phase 3-a
- FTS 검색/핀/자동추출 기본 동작

## Phase 3-b
- 하이브리드 검색(FTS+벡터+그래프) 동작

## Phase 4
- safeStorage 적용
- 구조화 로깅 확인
- 복구 스냅샷 동작 확인

## Phase 5
- 원격 인증/권한/타임아웃 정책 검증
