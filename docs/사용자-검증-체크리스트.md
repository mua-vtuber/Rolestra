# 사용자 검증 체크리스트 (dogfooding)

**대상 commit**: `abb8699` (2026-04-29)
**전제 조건**: cleanup F1~F6 (`96c5d4c`) + production bug B1~B8 (`abb8699`) 적용된 상태

이 문서는 사용자가 Windows PowerShell 에서 `npm run dev` 로 직접 띄워 자연 사용
흐름으로 따라가면서 점검하는 체크리스트다. 각 항목은 (i) **무엇을 한다**,
(ii) **어떤 결과를 본다**, (iii) **틀어졌을 때 보고할 정보** 의 3 단으로 구성.
"기대 결과" 와 다르면 해당 commit 의 회귀일 가능성이 크다.

## 0. 환경 준비

| 항목 | 동작 | 기대 |
|---|---|---|
| 0-1 | Windows PowerShell 에서 `npm install` (Linux/WSL 에서 한 번이라도 `npm rebuild` 했다면 Windows 바이너리 복구를 위해 필수) | electron / better-sqlite3 가 Windows 네이티브로 재컴파일됨 |
| 0-2 | `npm run typecheck` | 0 error |
| 0-3 | `npm run dev` | electron-vite dev 서버 + Electron 창. ArenaRoot 경로에 SQLite + 14 + **15** migration 적용 로그 (`015-approval-circuit-breaker-kind`). 콘솔 에러 없음 |

**migration 015 가 적용 안 보이면**: 기존 ArenaRoot DB 가 잠겨 있을 수 있음. 새 ArenaRoot
폴더로 첫 부팅 한 번 해 보면 분명히 적용된다.
