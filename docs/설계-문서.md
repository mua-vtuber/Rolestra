## 1. 개요

Rolestra 는 사용자가 여러 AI provider (Claude Code / Codex / Gemini CLI / Anthropic API / OpenAI API / Local Ollama) 를 **회사의 직원처럼** 운영하는 데스크톱 애플리케이션이다. 사용자가 `회사 (= ArenaRoot)` 를 차리고, 직원 (= AI provider) 을 고용하고, 프로젝트 (= 폴더) 를 만들고, 채널 (= 메신저 그룹) 에서 대화하고, 회의 (= MeetingSession) 로 합의를 도출하고, 승인 인박스 (= ApprovalInbox) 에서 결과를 승인하면 ExecutionService 가 atomic 하게 파일에 반영한다.

기존 v2 (`AI_Chat`, Python/FastAPI/Svelte 웹앱) 의 11-phase 재작성. 11 phase (R1~R11) 모두 단일 브랜치 + main fast-forward merge 패턴. R11 종료 시점에 v0.1.0 첫 사용자 출시.

### 1.1 Rolestra 메타포

| 메타포 | 코드 도메인 |
|--------|------------|
| 회사 | `ArenaRoot` (사용자 home 의 단일 디렉토리) |
| 직원 (AI) | `MemberProfile` + `Provider` |
| 프로젝트 | `Project` (3 kind: `new` / `external` / `imported`) |
| 채널 (메신저) | `Channel` (`kind`: `system` / `user` / `dm`) |
| 회의 | `MeetingSession` (participants ≥ 2, ConsensusStateMachine 구동) |
| 결재 인박스 | `ApprovalInbox` (5 kind discriminated union) |
| 큐 (자율 작업) | `QueuePanel` + `QueueService` |
| 출근 / 외근 / 점검 | `MemberStatus` (`online` / `offline` / `connecting` / `field_work` / `inspection`) |
| 회의록 | `meeting_minutes` (LLM summarize) |
| 결재함 | ApprovalInbox split layout — list + detail panel |

### 1.2 11-phase 구조

| Phase | 핵심 |
|-------|------|
| R1 | 폴더 접근 격리 스모크 (CA-6, ArenaRoot 매트릭스 검증) |
| R2 | v3 DB 스키마 + Main 레이어 9 서비스 + IPC 채널 |

| R6 | 회의 엔진 v3 재작성 (옵션 E — v2 engine 1,900 LOC 신규) |
| R7 | 승인 인박스 (5 kind discriminated union + cli/mode_transition/consensus 정식) |

## 3. 아키텍처 — 4 layer + 단방향 의존성

```
renderer ──→ shared ──→ preload (contextBridge) ──→ main
   ↑                                                   │
   │                                                   │
   └────────── stream:* events (one-way) ←─────────────┘
```

### 3.1 의존성 규칙 (CLAUDE.md 절대 위반 금지)

| 방향 | 허용 여부 |
|------|----------|
| `renderer` → `main` | ❌ 금지 (반드시 IPC 경유) |
| `renderer` → `preload` | ✅ contextBridge 노출 함수만 |
| `renderer` → `shared` | ✅ 타입/상수 |
| `main` → `renderer` | ❌ 금지 (one-way stream events 만) |
| `main` → `shared` | ✅ |
| `preload` → `shared` | ✅ |
| `preload` → `main` | ❌ (preload 는 contextBridge layer) |

## 4. 핵심 모듈

### 4.1 ConsensusStateMachine (12 state)

`src/main/meetings/engine/consensus-machine.ts`. 회의 합의 흐름의 단일 권위.

12 상태:
- `IDLE` → `DISCUSSING` → `SYNTHESIZING` → `VOTING` → `AWAITING_USER` → `APPLYING` → `DONE`
- 분기: `FAILED` / `EXPIRED` / `ABORTED` + 2 보조

설정:
- `aggregatorStrategy`: `strongest` (default) / `last-speaker` / `designated` / `round-robin`
- `maxRetries`, 이벤트, 가드, 타임아웃 명시
- 상태 스냅샷 저장 (`meetings.state_snapshot_json`) — 전이 시점 + 주기 체크포인트 + 종료 flush

10,000 LOC 테스트 자산 — R6+ 회귀 검증 근간 (ADR cross-cutting C1).

### 4.2 Provider Registry + Capability

`src/main/providers/registry.ts`. discriminated union.

```ts
type ProviderConfig =
  | { kind: 'api'; id: string; capabilities: ProviderCapability[]; ... }
  | { kind: 'cli'; id: string; capabilities: ProviderCapability[]; ... }
  | { kind: 'local'; id: string; capabilities: ProviderCapability[]; ... };

type ProviderCapability = 'streaming' | 'resume' | 'tools' | 'web-search' | 'summarize';
```

7 provider class (api 4: Anthropic / OpenAI / Google / OpenRouter, cli 3: Claude Code / Codex / Gemini, local 1: Ollama / llama.cpp / vLLM 3 변형). `'summarize'` capability 는 R11 Task 9 정식 (R10 임시 `'streaming'` 우회 제거).

### 4.3 ExecutionService 경계

`src/main/execution/execution-service.ts`. 모든 파일/명령 실행의 단일 진입점.

- `dryRun → 사용자 승인 → atomic apply → rollback` 순서 고정
- `CommandRequest` 구조화 — `execFile` + `shell: false` (셸 문자열 금지)
- `CommandPolicy` — `allowedCommands` / `blockedPatterns` / `maxExecutionTimeMs` / `maxOutputBytes`
- `AuditEntry` 영구 기록 — dry-run/apply/rollback 모두
- R11 Task 7 — `dryRunPreview(approvalId)` 추가 (read-only path, mutation 0 보장)

### 4.4 ArenaRootService + ProjectService

`src/main/arena/arena-root-service.ts` + `src/main/projects/project-service.ts`.

- ArenaRoot = 사용자 home 의 단일 디렉토리 (`~/Rolestra/` 또는 사용자 지정)
- 3 project kind:
  - `new` — `<ArenaRoot>/projects/<slug>/` 신규 폴더
  - `external` — `<ArenaRoot>/projects/<slug>/link` Windows junction / macOS+Linux symlink → 외부 경로
  - `imported` — 외부 폴더 자체를 ArenaRoot 안으로 복사 (R10+ 추가)
- junction TOCTOU 검증 — `path-guard.ts` 가 realpath 비교 (CA-3)
- 시스템 채널 자동 생성 — `ProjectService.create/linkExternal/importFolder` 가 `ChannelService.createSystemChannels` 호출 (R5 D2)

### 4.5 PermissionService + PermissionFlagBuilder

`src/main/files/permission-flag-builder.ts`. R10 Task 5 정식.

| autonomyMode | × | CLI kind | × | Project kind | = | flag set |
|--------------|---|----------|---|--------------|---|----------|
| `auto` | × | `claude` / `codex` / `gemini` | × | `new` / `external` / `imported` | = | `--dangerously-allow-all` 등 |
| `hybrid` | (동일) | (동일) | (동일) | turn-level approval gate |
| `approval` | (동일) | (동일) | (동일) | strict path allowlist + 매 turn 승인 |

39 cases (27 + 6 + 6) 단위 테스트 (R10 Task 5).

### 4.6 Memory (FTS5 — Phase 3-a)

`src/main/memory/`. SQLite FTS5 + 핀 + Regex.

- Phase 3-a (R3+) — FTS5 + 핀 + Regex 검색
- Phase 3-b (R12+ 이연) — 임베딩 + 하이브리드 + 반성 + 진화 (Stanford 3-factor scoring)

### 4.7 IPC TypedInvoke + zod

`src/preload/index.ts` + `src/shared/ipc-types.ts` + `src/shared/ipc-schemas.ts`.

```ts
function typedInvoke<K extends keyof IpcChannelMap>(
  channel: K,
  payload: IpcChannelMap[K]['request'],
): Promise<IpcChannelMap[K]['response']>;
```

- 모든 채널 — `IpcChannelMap` 의 generic key
- 개발 모드 — zod runtime validation (`v3ChannelSchemas` registry)
- production — 타입만 (성능)
- preload contextBridge 자동 화이트리스트 (R11 Task 5 검증)

R11 Task 5 land 시점 채널 수 — 60+ (R3 land 30 + R5/R6/R7/R8/R9/R10 추가 30 + R11 신규 8).

### 4.8 StreamBridge

`src/main/ipc/stream-bridge.ts`. one-way main → renderer push.

- `meeting:turn-token` / `meeting:turn-start` / `meeting:turn-done` / `meeting:state-changed` / `meeting:error`
- `stream:approval-created` / `stream:approval-decided`
- `stream:member-status-changed` (R10 Task 10 land + R11 D9 dual-path)
- `notification:show` (toast/native dual)

### 4.9 AutonomyGate + CircuitBreaker

`src/main/autonomy/`.

- `AutonomyGate` — ApprovalService 'created' 이벤트 훅 (R9 D1, ApprovalService 단일 책임 유지)
- `CircuitBreaker` — 4 tripwire:
  - `files_per_turn` (단일 턴 내 파일 수)
  - `cumulative_cli_ms` (CLI 누적 실행 시간)
  - `queue_streak` (queue 연속 실패)
  - `same_error` (동일 에러 연속)
- R10 Task 9 마이그레이션 012 — `circuit_breaker_state` (project_id, tripwire) PRIMARY KEY (D6)

## 10. 보안

### 10.1 절대 위반 금지 7 규칙 (CLAUDE.md)

1. Renderer 에서 Node API 직접 접근 금지 (IPC 경유)
2. 문자열 IPC 직접 호출 금지 (typedInvoke 만)
3. 승인 없는 파일 반영 금지 (ExecutionService 경유: dry-run → 승인 → apply → rollback)
4. 셸 문자열 실행 금지 (구조화 CommandRequest, execFile + shell: false)
5. 하드코딩 UI 문자열 금지 (모든 사용자 노출 문자열은 t() 경유)
6. API 키 평문 노출 금지 (secrets 계층 safeStorage)
7. 마이그레이션 파일 수정 금지 (forward-only, idempotent, 실패 시 앱 시작 차단)

### 10.2 secrets — safeStorage

- `provider_credentials` 컬럼은 `safeStorage.encryptString` 결과
- macOS Keychain / Windows DPAPI / Linux libsecret 자동 위임
- Renderer 는 "있음/없음" boolean 만 알 수 있음
- safeStorage 사용 불가 환경 (Linux 일부) 은 명시적 경고 + plain 저장 금지 (입력 거부)

### 10.3 path-guard — ArenaRoot 봉인

`src/main/files/path-guard.ts` 가 모든 쓰기 경로의 sentinel.
- 모든 쓰기 = `<ArenaRoot>/` 또는 `<ArenaRoot>/avatars/` 안 (R8 D2)
